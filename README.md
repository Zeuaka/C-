Также в классе реализована перегрузка вывода значений в строку, чтобы соответствовать условиям задачи

#### **Задача 3**  
Создайте сущность Имя, которая описывается тремя параметрами: Фамилия, Личное имя,

Отчество. Имя может быть приведено к строковому виду, включающему традиционное 

представление всех трех параметров: Фамилия Имя Отчество (например “Иванов Иван 

Иванович”). Необходимо предусмотреть возможность того, что какой-либо из параметров может 

быть не задан, и в этом случае он не учитывается при приведении к текстовому виду.

Необходимо создать следующие имена:

• Клеопатра

• Пушкин Александр Сергеевич

• Маяковский Владимир

Обратите внимание, что при выводе на экран, не заданные параметры никак не участвуют в 
образовании строки.

**Алгоритм работы:**  
Сущность реализована с помощью internal class Name, который предоставляет возможность как задать ФИО

передачей данных в конструктор с помощью клавиатуры, так и создать ее по умолчанию с фиксированными значениями ФИО.

Конструкторы класса дают возможность задать как ФИО, так и ФИ, ИО (задать имя для сущности обязательно)

Также в классе реализована перегрузка вывода значений в строку, чтобы соответствовать условиям задачи.

Так, например при отсутствии у сущности каких-либо атрибутов, будут выведены только те значения, которые у сущности есть.

### **Задание 2**

#### **Задача 1**  
Создайте сущность Линия, расположенную на двумерной плоскости, которая описывается:

• Координата начала: Точка

• Координата конца: Точка

• Может возвращать текстовое представление вида “Линия от {X1;Y1} до {X2;Y2}”

Для указания координат нужно использовать сущность Точка, разработанную в задании 1.1.


**Алгоритм работы:**  
Сущность реализована с помощью internal class MyLine. Программа предоставляет возможность задать координаты точек

с помощью клавиатуры, но так же в классе есть конструктор, который автоматически создает точки со случайными 

координатами для создания линии.

Класс реализован на логике двух указателей startPoint и endPoint, которые указывают на начальную и конечную точки.

Также в классе реализована перегрузка вывода значений в строку, чтобы соответствовать условиям задачи. 

В данный метод дополнительно встроена проверка на совпадение координат точек линии. 

Если линия начинается и заканчивается в одной и той же координате,

то такая линия будет считаться замкнутой

### **Задание 3**

#### **Задача 3**  
Создайте сущность Город, которая будет представлять собой точку на карте со следующими

характеристиками:

• Название города

• Набор путей к следующим городам, где путь представляет собой сочетание Города и 

стоимости поездки в него.

Кроме того, Город может возвращать текстовое представление, в виде названия города и списка 

связанных с ним городов (в виде пары: “название:стоимость”).

**Алгоритм работы:**  
Сущность реализована с помощью internal class City.

Класс включает в себя string name в который будет передаваться название города, а также два массива:

City[] neighbourCities в который будут передаваться города, в которые будут проходить дороги

и int[] roadCosts в который будут передаваться стоимости переезда в эти города.

Также, раз мы работаем с массивами, чтобы избежать выхода за границы массивов, нам необходимо ввести счетчик дорог roadsCount

Задать город в программе можно просто указав его название и передав его в конструктор класса.

Дороги к городам добавляются с помощью метода AddRoad, в который параметром передается уже существующий город и цена дороги к нему.

При успешной работе метода, город передается в массив с городами neighbourCities, а цена дороги в roadCosts.

Также в классе реализована перегрузка вывода значений в строку, 

Так, при вызове Console.Writeline(Город), Произойдет проход циклом от 0 до счетчика roadCount, 

во время которого по индексу будут извлекаться данные из массивов neighbourCities и roadCosts, и выводиться в консоль.

### **Задание 5**  

#### **Задача 5**  
Создайте сущность Дробь со следующими особенностями:

• Имеет числитель: целое число

• Имеет знаменатель: целое число

• Дробь может быть создана с указанием числителя и знаменателя 

• Может вернуть строковое представление вида “числитель/знаменатель”

• Может выполнять операции сложения, вычитания, умножения и деления с другой Дробью 

или целым числом. Результатом операции должна быть новая Дробь (таким образом,обе 

исходные дроби не изменяются) 

**Алгоритм работы:**  
Сущность реализована с помощью internal class Fraction.

Инициализация дроби:

При создании дроби проверяется, что знаменатель не равен нулю

Если знаменатель равен нулю, выводится сообщение об ошибке и дробь устанавливается в 0/1

После создания дробь автоматически упрощается методом ToSimple()

### Упрощение дроби (метод ToSimple):

Если числитель равен 0, знаменатель устанавливается в 1

Вычисляется наибольший общий делитель (НОД) числителя и знаменателя с помощью алгоритма Евклида

Числитель и знаменатель делятся на НОД

Если знаменатель отрицательный, знак переносится на числитель

### Математические операции:

Все операции (Sum, Minus, Multiply, Div) возвращают новую дробь, не изменяя исходные

Операции с целыми числами преобразуют число в дробь (число/1)

При делении проверяется деление на ноль

Перегруженные операторы:

Поддерживаются операции +, -, *, /
