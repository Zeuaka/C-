### **Задание 1**

#### **Задача 1**  
Первый массив, размерностью n х m, заполняется данными, вводимыми с клавиатуры, так что 

заполнение ведется по строкам от последних элементов строки к первым.

Второй массив, размерностью n х n, заполняется так: элементы, лежащие ниже главной 

диагонали, являются случайными числами из интервала [-17;36], а лежащие на главной 

диагонали и выше, являются случайными числами из интервала [100; 10000]. 

Третий массив, размерностью n х n, заполняется для произвольного n так же, как для n=5:

<img width="257" height="154" alt="image" src="https://github.com/user-attachments/assets/625bfbe5-967c-41d0-815f-0e6e8cea8595" />


**Алгоритм работы:**  
Для решения задачи реализован класс Matix и три конструктора для каждого случая входных данных:

1) Для первого случая задачи в конструкторе по введенным пользователем размерам m и n создаается матрица.

Далее происходит проход по всем ее элементам, и запрашиваются элементы на ввод склавиатуры,

с дальнейшим присваиванием последних к элементам матрицы.


2) В конструкторе для второго случая запрашивается только одно значение n с клавиатуры, т.к. матрица квадратная,

генерация происходит с помощью random, с проверкой условия j<i, что и будет указателем, в какой плоскости относительно

главной диагонали находится элемент.

3) В конструкторе для этого случая, в отличие от предыдущего так же добавлен булевый идентификатор, чтобы отличать

этот конструктор от предыдущего

Алгоритм заполнения:

Матрица заполняется числами по диагональным линиям, параллельным побочной диагонали

Заполнение начинается с правого нижнего угла и движется вверх-влево по диагонали

Числа записываются в убывающем порядке, начиная с максимального значения

Когда достигается граница матрицы, происходит переход к следующей диагонали

### **Задание 2**

#### **Задача 1**  
Найдите максимальный из неповторяющихся (встречающихся только один раз) элементов 

двумерного массива.


**Алгоритм работы:**  
В программе реализован выбор типа заполнения матрицы (опции такие же, как в 1 задании).

Поиск уникального максимума реализован следующим образом:

Первым проходом цикла все элементы матрицы переводятся в о одномерный массив, количество

ячеек в котором = n*m.

Далее в массиве такой же размерности с помощью второго прохода по матрице пересчитываем все повторения

этих элементов и записываем результаты измерений в массив.

После чего, уже третьим проходом цикла уже массиву элементов ищется максимум, с проверкой, что этот элемент

встречался только единожды

### **Задание 4**

#### **Задача 1**  
Получить в новом файле те компоненты исходного файла, которые являются четными.

**Алгоритм работы:**  

В начале программы файл заполняется случайными числами в количестве 10 штук.

Далее в классе для работы с файлами реализуются BinaryWriter и BinaryReader,

И пока не достигается конец файла, считываются элементы входного файла по формату Int32,

тоесть по 4 байта. Если элемент оказывается четным, он записывается в конечный файл в бинарном виде.

### **Задание 5**  

#### **Задача 1**  
Информация о багаже пассажира описывается массивом, где каждый элемент содержит название

единицы багажа (чемодан, сумка, коробка и т.д.) и ее массу. Дан файл, содержащий сведения о

багаже нескольких пассажиров. Найти багаж, средняя масса одной единицы багажа, в котором

отличается не более чем на m кг от общей средней массы одной единицы багажа.


**Алгоритм работы:**  

### 1. Подготовка тестовых данных

Создается объект LuggageData с массивом пассажиров

Для каждого пассажира задается:

Имя пассажира

Массив единиц багажа (название + вес)

Пример данных:

Иванов: Чемодан(15.5кг), Рюкзак(3.2кг), Сумка(4.1кг)

Петров: Чемодан(12.8кг), Коробка(6.7кг)

Сидоров: Рюкзак(5.3кг), Сумка(2.9кг), Пакет(1.2кг)

Данные сохраняются в XML файл luggage_data.xml

### 2. Загрузка данных из XML

Используется XmlSerializer для десериализации данных

Восстанавливается структура объектов:

LuggageData → массив PassengerLuggage

Каждый PassengerLuggage → массив LuggageItem

### 3. Расчет общей статистики

Шаги расчета:

Вычисление общего суммарного веса и количества единиц

Расчет общей средней массы (общий вес / количество единиц)

Поиск подходящего багажа (который удовлетворяет среднему допустимому отклонению от среднего веса)

### **Задание 6**  

#### **Задача 1**  
Количество элементов файла чётно. Определить разность суммы элементов первой и второй 
половины файла.

**Алгоритм работы:**  
Сначала файл заполняется построчно n случайными числами (n вводится с клавиатуры).

Далее, осуществляется считывание всех строк файла, и записывается в массив строк lines.

После чего, считывается длина массива, и осуществляется проход с сумированием элементов первой его половины.

Наконец проводится такая же операция для его второй половины, и вычисляется разность полученных сумм элементов.

### **Задание 8**  

#### **Задача 1**  
Переписать в другой текстовый файл самую короткую и самую длинную строки.

**Алгоритм работы:**  
Для начала файл заполняется строками (в моем случае это реализовано с помощью метода).

После чего строки файла считываются в массив строк, а также создаются два массива с одной ячейкой,

В которую записывается первая строка массива строк.

Далее происходит проход по всем строкам массива, и сравниваются длины строк с длинами сртоки, записанной в каждый из введенных масивов

(один из этих массивов должен хранить самую большую строку, а другой самую маленькую).

В конце прохода, получившиеся массивы записываются в новый файл
